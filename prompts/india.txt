SECTION 2 — OCR ERROR PATTERNS ON OLD SOI MAPS
═══════════════════════════════════════════════════════

CHARACTER CONFUSIONS (common on aged ink / faded print):
  rn → m      ("Rarnpur" → "Rampur", "Barnala" is correct not "Bam ala")
  ii → u/n    ("Naiini" → "Naini",  "Delhii" → "Delhi")
  vv → w      ("Nevv" → "New",  "Navvab" → "Nawab")
  8 → B       ("8ijnor" → "Bijnor",  "8engal" → "Bengal")
  6 → G       ("6anga" → "Ganga",  "6aya" → "Gaya")
  0 → O       ("0udhpur" → "Oudh pur → Oudh",  "0orakhpur" → "Gorakhpur")
  1 → I / l   ("1ndia" → "India",  "Ba1ly" → "Bally")
  c → e       ("Allahubad" → "Allahabad")
  broken space → rejoin:
    "Allaha bad" → "Allahabad"
    "Pala mau"   → "Palamau"
    "Haza ri bagh" → "Hazaribagh"
    "Muz affarpur" → "Muzaffarpur"
    "Cham paran" → "Champaran"
  dot-above → ignored:  "Māhābhārata" → "Mahabharata"
  faded final consonant: "Patna" misread as "Patm" or "Patn"

SCRIPT-SPECIFIC ISSUES:
  Urdu-origin words often have silent consonants: "Khwaja" → OCR may give "Khwaga"
  Persian/Urdu: kh, gh, sh, ch clusters may be split by OCR
  Old Gothic/italic print (pre-1900 maps): f → s, long-s → f, ct → df

ABBREVIATIONS ON MAP MARGINS:
  "Surv." or "Surveyed by …"   (top margin, NOT a feature)
  "Pubd." = Published
  "Coy." = Company (as in East India Company era maps)
  "Engd." = Engraved
  "Sheet No." / "No." = sheet number reference (NOT a place name)
  "Corrd." = Corrected
  Scale bar text: "Miles", "Furlongs", "Chains", "Yards", "Kilometres"
  "R.F. 1/63360" = Representative Fraction (NOT a place name)
  "Restricted" / "Open Survey Sheet" / "Confidential" (security classification, ignore)
  "Magnetic Declination" / "Variation" text (NOT a feature)
  "Contour interval … feet/metres" (NOT a feature)

SECTION 3 — GRID SYSTEM ON SOI SHEETS
═══════════════════════════════════════════════════════

IMPORTANT — DO NOT confuse these with grid labels:
  Large decorative area-name letters printed across the map body
  (e.g. "P U R H A T", "B E N G A L", "B I H A R" spanning the map)
  are the DISTRICT/REGION NAME, not grid identifiers.
  True grid labels are SMALL letters/numbers printed ONLY in the margin area.

GRID COLUMN LABELS (in top + bottom margin):   A  B  C  D
GRID ROW LABELS    (in left + right margin):   1  2  3  4
→ Cell reference = letter + hyphen + number, e.g. "B-3"

Old 1:63,360 (1-inch) maps may show a 4×4 grid (16 cells) or a 3×4 grid (12 cells).
1:50,000 maps consistently show a 4×4 grid (16 cells, labelled A–D × 1–4).

SECTION 6 — ZENODO / JOHN BROWN COLLECTION FILENAME FORMAT
═══════════════════════════════════════════════════════

Files from https://zenodo.org/records/8388147 follow this pattern:
  "<block> <letter>]<num> <District> (<year>)[optional].jpg"
  e.g.  "72 P]16 Birbhum District (1924).jpg"
        "72 B]04 Siwan District (1974) BW@.jpg"
        "72 F]02 and F[06 Champaran District (1915).jpg"

CODES in filename:
  BW = Black & White photographic scan (lower contrast)
  @  = Damaged / partial sheet
  Preliminary = early, un-revised survey
  35MM-FILMED = microfilm reproduction (may have vignetting)

SHEET NUMBERING (SOI standard):
  Block = 1° × 1° square (e.g. Block 72 = approx. 23–24°N, 86–87°E)
  Letter = A–P (skipping I), sub-divides block into 4 columns × 4 rows
  Number = 1–16, sub-divides the letter block into a 4×4 internal grid
  Full ref: "72P/16" means block 72, letter P, sub-sheet 16
